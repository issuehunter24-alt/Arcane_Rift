## 카드 데이터 사양

### 1. 스키마 필드
- id: 고유 식별자(문자열)
- name: 표시 이름(현지화 키 사용 가능)
- type: Attack | Heal | Special | Defense
- rarity: Normal | Rare | Epic | Legendary
- cost: 정수
- effectText: 짧은 설명(키워드 강조)
- tags: 속성/시너지 태그 배열(불/얼음/번개/바람 등)
- keywords: 기능 키워드 배열(화상/빙결/감전/무효/취약 등)
- levelCurve: { base, perLevel }
- vfxKey, sfxKey: 연출 키
- version: 데이터 버전(마이그레이션 기준)

### 2. 키-리스트 매핑
- 소스: `카드/CardList_Formatted.md` → 변환 파이프라인(JSON).
- 규칙: 동일 name이라도 id로 판정, 버전 필드로 변경 추적.

### 3. 표준 규칙
- 문자열 길이 가이드: 이름 18자 이내, effectText 2줄 내 요약.
- 비용·효과 표준화 테이블에 따라 수치 검증(밸런스 파이프라인).

### 4. 예시(JSON)
```
{
  "id": "ATK_FIREBALL_001",
  "name": "파이어볼",
  "type": "Attack",
  "rarity": "Rare",
  "cost": 2,
  "effectText": "적에게 피해 35. 화상 1중첩.",
  "tags": ["Fire"],
  "keywords": ["Burn"],
  "levelCurve": { "base": {"damage": 35}, "perLevel": {"damage": 3} },
  "vfxKey": "atk_fire",
  "sfxKey": "hit_fire",
  "version": 1
}
```

### 5. 제약/검증 규칙(일관성)
- id 네이밍: `<TYPE>_<THEME>_<INDEX>` 대문자 스네이크(예: ATK_FIREBALL_001). 전역 유일.
- type↔keywords 일관성: Attack에만 직접 피해 수치, Heal에만 회복 수치 기본 포함. Special/Defense는 직접 피해 최소화.
- rarity 한도: Normal은 복합 키워드 1개 이하, Legendary는 조건부 강화 허용.
- cost 범위: 0~5(초기안). 0코스트는 소진 또는 하드 제약 필요.
- effectText: 키워드 영문 표기는 툴팁 자동 치환(문서 ‘키워드_정의서’ 참조).
- tags 목록: Fire/Ice/Lightning/Wind 외 확장 시 표준 문서에 선등록.
- vfxKey/sfxKey: ‘오디오_이펙트_가이드’ 매핑 테이블에 존재해야 함.

### 6. 버전/마이그레이션
- version 증가 시 변경 이력 기록(변경 필드, 사유, 날짜).
- 세이브 호환: 동일 id 카드의 수치 변경은 즉시 반영, 삭제는 ‘대체 id’ 매핑 제공.

### 7. 데이터 파이프라인/검증
- 변환: Markdown → JSON 변환 스크립트. 필드 스키마 검증(JsonSchema).
- 밸런스 체크: 기대값 룰(문서 ‘경제_드랍_상점_정책’)로 자동 경고.
- 로컬라이제이션: name/effectText는 키 기반 번역 테이블로 분리 가능.

### 8. 키워드 종류(예상 범위 명시)
- 총량 목표: MVP 8~12종 → 확장 12~16종 → 라이브 16~20종 내 관리.
- MVP(출시 최소): 8~12
  - 상태/디버프: Burn, Freeze, Shock, Vulnerable
  - 방어/보호: Shield, Guard
  - 대응/관리: Nullify, Cleanse
- 확장 1차(M2~M3): 12~16
  - Buff, Weaken, Regen, Thorns, Silence 등
- 라이브(메타 다양화): 16~20
  - Haste, Slow, Lifesteal, Counter, Dispel 등
- 분류 상한 가이드(학습/밸런스 비용 관리)
  - 상태/도트/제어: 6~8, 방어/보호/저항: 4~6, 버프/디버프: 6~8, 대응/반응: 3~4
- 원칙: 새 키워드는 “새 타이밍/자원/영역” 도입 시에만 추가. 그 외는 기존 프리미티브 조합/파라미터로 해결.
