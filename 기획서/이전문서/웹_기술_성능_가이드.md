## 웹 기술/성능 가이드

### 1. 렌더/스택
- TypeScript + PixiJS(또는 Phaser), Vite 번들, Zustand/Redux 상태.

### 2. 반응형/해상도
- 기준 1536×1024, 카드 768×1024 스케일. 모바일 브레이크포인트 ≤900px.

### 3. 에셋/로딩
- PNG → WebP/AVIF(품질 85) 변환, 스프라이트 아틀라스, LQIP → 지연 로딩.
- CDN 해시 버전, 캐시 무효화 규칙.

### 4. 성능 예산
- 60fps 목표(모바일 30fps 폴백), VRAM 상한(모바일 256MB 초안).
- 드로우콜 최소화, 비가시 텍스처 언로드.

### 5. 로딩 전략 상세
- 초기 번들 ≤1.0MB, 나머지 지연 로딩(전투/덱 편집 별도 청크).
- 프리로드: 메인 UI 폰트/프레임/아이콘, LQIP로 카드 미리보기.
- 인전트 로딩: 전투 입장 직후 필요한 카드 텍스처 선로딩.

### 6. 디바이스 타겟/폴백
- 데스크톱: 1080p 기준, DPR 1~2. 모바일: 720p~1080p 스케일.
- WebGL 실패 시 2D 캔버스 폴백(효과 단순화), 애니메이션 기간 축소.

### 7. 품질/안정성
- FPS/메모리 HUD 디버그 토글, 성능 로그 주기적 샘플링.
- 크래시 로깅(콘솔 에러/리소스 로드 실패) 수집 및 리포트.
